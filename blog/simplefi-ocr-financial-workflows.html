SimpleFi: OCR and financial workflow automation

June 2026 / Cyril Blyseth

At SimpleFi, I worked on OCR, underwriting automation, QA, interfaces, databases, and financial analysis workflows. A lot of the work was about turning messy documents into usable financial workflow data.

What I built and worked on

  • OCR automation for bank statements and pay stubs.
  • Training and filtering work to improve document extraction.
  • Identity, income, expense, and cash-flow calculations for underwriting-style review.
  • Automation around onboarding and financial analysis workflows.
  • A budgeting application/site that classified and clustered spending from bank statements.
  • Logic for money coming in, money going out, recurring bills, spending categories, and percentages.
  • QA, bug fixing, interfaces, database-connected workflows, and Spring/MVC platform work.
  • Translation between user needs, loan officer workflows, and developer implementation.

The technical shape

The workflow looked roughly like this: 

bank statements / pay stubs
  -> OCR
  -> cleanup and filtering
  -> extracted fields
  -> identity / income / expense / cash-flow calculations
  -> categorized spending
  -> review workflow

This was years before LLMs became the default answer for everything, but it hit a lot of the same problems I still care about: noisy input, incomplete data, automation that has to be reviewed by people, and workflows that need to make the next step easier.

What it proves

  • I have been building practical automation around messy real-world data for a long time.
  • I am comfortable moving between code, QA, workflow design, databases, and user-facing requirements.
  • The AI work I care about now is not a sudden pivot. It connects back to OCR, classification, extraction, and process automation.
